Hébergement VST3

Utilisez des instruments et effets tiers dans le rack GrooveForge sur ordinateur.

Plateformes prises en charge (aujourd’hui)

Linux macOS — Hébergement VST3 complet : chargez des bundles .vst3, traitez l’audio dans le même graphe patch que les modules intégrés, ouvrez l’éditeur natif de chaque plugin dans une fenêtre séparée (X11 sous Linux ; toolkit natif sous macOS). BPM et transport sont transmis pour garder les effets synchronisés au tempo alignés sur GrooveForge.

Windows : la plomberie existe dans le code mais l’intégration audio WASAPI est encore en cours — considérez le VST3 Windows comme prévu jusqu’à une annonce dans le journal des modifications.

Web / Android — le VST3 n’est pas disponible ; utilisez le clavier GrooveForge, les effets .gfpd fournis et les MIDI FX.

Charger un plugin

  1. Touchez + dans le rack → Parcourir les VST3 (bureau uniquement).
  2. Sélectionnez le dossier ou fichier du bundle .vst3 (sélecteur dépendant de l’OS).
  3. Les paramètres apparaissent en potentiomètres rotatifs groupés dans la carte du slot ; Afficher l’interface du plugin ouvre la fenêtre de l’éditeur.

Routage audio

Dans la vue patch, reliez la sortie audio d’un clavier ou d’un autre plugin à l’entrée audio d’un VST, puis vers le bus principal ou d’autres effets. L’ordre suit le graphe de câbles (traitement topologique). Enregistrez le projet pour capturer à la fois le rack et les instantanés de paramètres VST dans le .gf.

Prérequis de compilation

Le SDK VST3 de Steinberg n’est pas inclus dans le dépôt. Clonez-le dans packages/flutter_vst3/vst3sdk (avec sous-modules) avant de compiler les cibles bureau — voir le README sur GitHub pour la commande exacte.
